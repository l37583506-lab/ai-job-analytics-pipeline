# 基于 RAG 与关键词特征工程的招聘数据分析 Pipeline

这是一个针对 1.2 万条 LinkedIn 原始招聘数据的数据处理与智能检索项目。项目实现了离线端的数据清洗、多维度标签抽取（岗位角色/技术栈/资深职级）以及基于向量空间模型的语义检索引擎。

## 核心技术点
- **数据清洗与异常处理**：编写正则算子批量剔除网页残留 HTML 标签与多余空白符；通过类型检查（`isinstance`）拦截处理 Pandas 读取产生的 `NaN` 空值，确保 1.2 万行长尾文本的 Pipeline 运行健壮性。
- **特征规约与标签映射 (NER)**：针对非结构化的职位文本进行分词与关键词匹配，将无序文本标准化规约为核心角色（Role）、技术栈（Tech Stack，支持多标签）以及级别（Level）三个维度，完成非结构化数据到结构化特征表的转化。
- **向量空间检索 (RAG 召回)**：离线端利用 TF-IDF 算法将文本转化为稀疏向量矩阵；在线端采用**余弦相似度（Cosine Similarity）**算法计算查询词（Query）与知识库的高维空间夹角，通过矩阵降序切片实现 Top-K 语义搜索匹配。

## 数据洞察分析（样本量：12,216）
- **资深岗位主导**：高级别岗位（Senior）市场占比达 **28.71%**，表明当前市场对具备成熟经验的人才需求最高。
- **数据基建需求大于模型开发**：数据工程与数据基础设施类岗位（**32.07%**）需求量远超纯 ML/AI 算法模型岗，企业在落地 AI 应用时明显更优先进行数据管道建设。

## 技术栈
- 语言：Python 3.10+
- 数据处理：Pandas, NumPy
- 文本处理：`re` (正则表达式)
- 机器学习/检索：Scikit-learn (TfidfVectorizer, Cosine Similarity)
